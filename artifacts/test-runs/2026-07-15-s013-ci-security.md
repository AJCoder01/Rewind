# S013 CI and repository security evidence — 2026-07-15

## Scope

S013 adds a locked Node 24 GitHub Actions job covering install, lint, strict typecheck, unit/contract tests, production build, dependency audit, reachable-history secret scanning, production fake-mode rejection, ephemeral PostgreSQL migration apply/replay, and the fixture browser smoke. Checkout uses full history and immutable action SHAs.

The scan reads tracked working-tree files plus reachable committed blobs. It reports only file/rule identifiers and never prints matched content. A tracked or historical private filename fails without its content being read. CI's PostgreSQL service is an isolated loopback-only `rewind_ci` test service; the migration helper requires `CI=true` and the fixed fake URL shape before it can create a pool. It is not a provider or demo environment.

## Correction pass

The original hosted runs failed before PostgreSQL/E2E execution because the foundation migration hash used Windows CRLF bytes while GitHub checks out LF bytes. The runner now canonicalizes line endings and pins the LF hash. It recognizes only the previously recorded CRLF hash for the already-reviewed migration ledger, without rewrite or a non-disposable database connection. The fixture browser server starts with an explicit `NODE_ENV=test` fixture environment and never inherits caller/deployment variables.
